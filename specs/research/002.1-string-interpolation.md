# Story 002.1: String Interpolation Syntax Research

**Date**: 2026-05-15  
**Status**: ✅ Complete

## Research Question

What syntax should LULCODE use for string interpolation?

## Key Finding

**LOLCODE 1.2 ALREADY HAS variable interpolation!**

Syntax: `:{VAR}:`

Example:
```lolcode
I HAS A name ITZ "Alice"
VISIBLE "Hello :{name}:"  ← Outputs: Hello Alice
```

From spec: ":{`<var>`}" interpolates variable values as strings (automatic YARN cast)

## Syntax Options for LULCODE

### Option 1: Use LOLCODE Native (No Sugar)
```lulcode
"Hello :{name}:"  ← Pass through as-is
```

**Pros**:
- ✅ Already works
- ✅ No transpilation needed
- ✅ 100% backward compatible
- ✅ Zero implementation effort

**Cons**:
- ❌ Colons are unusual/less readable
- ❌ No improvement over LOLCODE

### Option 2: Add Modern Sugar
```lulcode
"Hello {name}"  → transpiles to → "Hello :{name}:"
```

**Pros**:
- ✅ Familiar to modern developers
- ✅ Cleaner syntax (no colons)
- ✅ Matches JavaScript, Python f-strings, etc.
- ✅ Can still use `:{...}:` for backward compat

**Cons**:
- ❌ Requires transpilation
- ❌ Need to handle literal braces (escaping)
- ❌ More complex parser

### Option 3: Extend LOLCODE Syntax (Alternative)
```lulcode
"Hello ${name}"  → transpiles to → "Hello :{name}:"
```

**Pros**:
- ✅ Familiar (shell, JS template literals)
- ✅ `$` signals variable clearly
- ✅ Less conflict with literal braces

**Cons**:
- ❌ Still requires transpilation
- ❌ Less clean than plain braces

## The SMOOSH Problem

Current game.lol:
```lolcode
VISIBLE SMOOSH "WIRE JOIN " NAME MKAY
VISIBLE SMOOSH "STATE revealed=" RTXT " votes=" VOTES MKAY
```

With LOLCODE native syntax:
```lolcode
VISIBLE "WIRE JOIN :{NAME}:"
VISIBLE "STATE revealed=:{RTXT}: votes=:{VOTES}:"
```

With LULCODE sugar:
```lulcode
VISIBLE "WIRE JOIN {NAME}"
VISIBLE "STATE revealed={RTXT} votes={VOTES}"
```

**Impact**: Both eliminate SMOOSH! Native syntax already solves the problem.

## Literal Brace Handling

If we add `{...}` syntax, how to escape literal braces?

**Options**:
- `\{` and `\}` (backslash escape)
- `{{` and `}}` (double braces, like Python f-strings)
- `:{` and `}:` (use LOLCODE colon escapes)

## Recommendation

**Option 2: Add `{VAR}` sugar, transpile to `:{VAR}:`**

**Rationale**:
1. Solves SMOOSH problem completely ✓
2. Modern, readable syntax ✓
3. Backward compatible (can still use `:{...}:`) ✓
4. Matches user expectations ✓
5. Implementation straightforward (regex replacement)

**Implementation**:
```javascript
// In transform.js
output = output.replace(
  /"([^"]*\{[^}]+\}[^"]*)"/g,  // Match strings with {var}
  (match, content) => {
    // Replace {var} with :{var}:
    const interpolated = content.replace(/\{(\w+)\}/g, ':{$1}:');
    return `"${interpolated}"`;
  }
);
```

## Escape Literal Braces

Use double braces (like Python):
- `{{` → `{`
- `}}` → `}`

Example:
```lulcode
"Use {{braces}} like {this}"  
→ "Use {braces} like :{this}:"
```

## Backward Compatibility

LULCODE strings can use:
- `{var}` - LULCODE sugar (new)
- `:{var}:` - LOLCODE native (still works)
- Both can coexist in same file
