# Story 003.4: Loop Syntax Shorthand - Research & Design

**Status**: In Progress  
**Date**: 2026-05-19

## Overview

Design cleaner loop syntax, especially **FOREACH for BUKKIT iteration**. Current LOLCODE loops are verbose and BUKKIT iteration is nearly impossible without arrays.

## Current LOLCODE Loop Syntax

### Basic Loop (Increment)
```lolcode
IM IN YR loop UPPIN YR i TIL BOTH SAEM i AN 10
  VISIBLE i
IM OUTTA YR loop
```

**Extremely verbose** for a simple 0-to-10 loop!

### Decrement Loop
```lolcode
IM IN YR loop NERFIN YR i TIL BOTH SAEM i AN 0
  VISIBLE i
IM OUTTA YR loop
```

### With Function (Iterator)
```lolcode
IM IN YR loop UPPIN YR i TIL I IZ done YR i MKAY
  BTW loop body
IM OUTTA YR loop
```

### Infinite Loop
```lolcode
IM IN YR loop
  BTW infinite
  GTFO                    BTW Break
IM OUTTA YR loop
```

## Problem: BUKKIT Iteration Impossible

BUKKIT has no native iteration! Must manually track keys:
```lolcode
BTW How do you iterate a BUKKIT???
I HAS A player ITZ A BUKKIT
player HAS A name ITZ "Alice"
player HAS A score ITZ 100

BTW Can't do:
BTW   for each key in player...
BTW   for each property...

BTW Best you can do: Know keys in advance
I HAS A pName ITZ player'Z name
I HAS A pScore ITZ player'Z score
```

**This is a major limitation!**

## Design Goals

1. **FOREACH for BUKKIT**: Essential for usability
2. **Cleaner numeric loops**: Shorter FOR syntax
3. **WHILE loops**: Condition-based iteration
4. **Backward compatible**: IM IN YR still works
5. **Maintain LOLCODE option**: Not too generic

## Part 1: FOREACH for BUKKIT

### Option 1: Python-style FOR IN
```lulcode
FOR key IN bukkit
  VISIBLE "Key: {key}"
END

FOR key, value IN bukkit
  VISIBLE "{key} = {value}"
END
```

**Pros**:
- Universal syntax (Python, JavaScript, Ruby)
- Clean and concise
- Key and value extraction

**Cons**:
- Very generic (not LOLCODE-y)
- "FOR" conflicts with numeric FOR loops

### Option 2: FOREACH keyword
```lulcode
FOREACH key IN bukkit
  VISIBLE "Key: {key}"
END

FOREACH key, value IN bukkit
  VISIBLE "{key} = {value}"
END
```

**Pros**:
- Distinct from numeric FOR
- Clear iteration intent
- Common in many languages (PHP, C#, Java)

**Cons**:
- Longer than FOR
- Still generic

### Option 3: LOLCODE-style IM IN YR
```lulcode
IM IN YR bukkit FINDIN BOTH key AN value
  VISIBLE "{key} = {value}"
IM OUTTA YR bukkit
```

**Pros**:
- Preserves LOLCODE flavor
- Consistent with existing loop syntax

**Cons**:
- Verbose (no improvement)
- Awkward syntax

### Recommendation: **FOREACH keyword**

```lulcode
FOREACH key IN bukkit
  BTW iterate with keys only
END

FOREACH key, value IN bukkit
  BTW iterate with keys and values
END
```

**Rationale**:
- Distinct from numeric loops (FOR vs FOREACH)
- Clear iteration purpose
- Familiar to many languages
- Concise enough for practical use

### FOREACH Transpilation Challenge

**Problem**: LOLCODE has no native BUKKIT iteration!

**Solution**: Generate loop with manual key tracking

**Approach 1**: Require user to maintain key list
```lolcode
BTW User must create keys BUKKIT:
I HAS A keys ITZ A BUKKIT
keys HAS A 0 ITZ "name"
keys HAS A 1 ITZ "score"

BTW Then iterate:
IM IN YR loop UPPIN YR i TIL BOTH SAEM i AN 2
  I HAS A key ITZ keys'Z SRS i
  I HAS A value ITZ player'Z SRS key
  BTW use key and value
IM OUTTA YR loop
```

**Approach 2**: Convention-based key tracking
```lulcode
BTW LULCODE generates key tracking:
BTW User creates: player HAS A name, player HAS A score
BTW LULCODE maintains: player_KEYS ITZ ["name", "score"]

BTW Then FOREACH transpiles to:
IM IN YR loop UPPIN YR i TIL BOTH SAEM i AN LENGZ OF player_KEYS
  I HAS A key ITZ player_KEYS'Z SRS i
  I HAS A value ITZ player'Z SRS key
IM OUTTA YR loop
```

**Approach 3**: Runtime library function
```lolcode
BTW LULCODE provides:
HOW IZ I __LULCODE_BUKKIT_KEYS YR bukkit
  BTW Return array of keys (somehow?)
  BTW Problem: No way to enumerate properties!
IF U SAY SO
```

**Problem**: LCI doesn't provide property enumeration!

### FOREACH Decision: **Deferred Pending Solution**

FOREACH requires either:
1. Arrays (EPIC-006) to store keys
2. LCI extension for property enumeration
3. Convention where user maintains keys manually

**For MVP**: 
- Design FOREACH syntax now
- Document transpilation strategy
- Implementation deferred until arrays available (EPIC-006)

**Alternative**: User-maintained keys convention
```lulcode
VAR players ITZ A BUKKIT
VAR player_keys ITZ A BUKKIT  BTW Parallel keys structure

BTW When adding to BUKKIT:
players["Alice"] = A BUKKIT
player_keys["0"] = "Alice"    BTW Track key

BTW FOREACH transpiles to manual iteration:
FOREACH key IN players
  BTW Uses player_keys
END
```

## Part 2: Numeric FOR Loops

### Option 1: FOR...TO...END (BASIC-style)
```lulcode
FOR i FROM 0 TO 10
  VISIBLE i
END

FOR i FROM 10 TO 0 STEP -1
  VISIBLE i
END
```

**Pros**:
- Clear range specification
- STEP for custom increment
- Familiar to BASIC programmers

**Cons**:
- Verbose (FROM/TO/STEP)
- Less compact than alternatives

### Option 2: FOR...IN RANGE (Python-style)
```lulcode
FOR i IN RANGE 0, 10
  VISIBLE i
END

FOR i IN RANGE 10
  BTW 0 to 9
END
```

**Pros**:
- Compact
- Python-style familiar
- RANGE clear meaning

**Cons**:
- Requires RANGE function/keyword
- Less explicit than FROM/TO

### Option 3: Shortened IM IN YR
```lulcode
LOOP i FROM 0 TO 10
  VISIBLE i
END
```

**Pros**:
- Shorter than IM IN YR
- Still English-readable

**Cons**:
- LOOP generic (infinite vs counted?)

### Recommendation: **FOR...FROM...TO...END**

```lulcode
FOR i FROM 0 TO 10
  VISIBLE i
END

FOR i FROM 0 TO 10 STEP 2
  VISIBLE i                   BTW 0, 2, 4, 6, 8, 10
END

FOR i FROM 10 TO 0 STEP -1
  VISIBLE i                   BTW Countdown
END
```

**Transpilation**:
```lolcode
i R 0
IM IN YR loop UPPIN YR i TIL BOTH SAEM i AN 10
  VISIBLE i
IM OUTTA YR loop
```

With STEP:
```lolcode
i R 0
IM IN YR loop
  BOTH SAEM i AN BIGGR OF i AN 10, O RLY?
    YA RLY
      GTFO
  OIC
  VISIBLE i
  i R SUM OF i AN 2
IM OUTTA YR loop
```

## Part 3: WHILE Loops

### Option 1: WHILE...END
```lulcode
WHILE condition
  BTW loop body
END
```

**Pros**:
- Universal syntax
- Clear condition-based loop
- Most concise

**Cons**:
- Generic (not LOLCODE-y)

### Option 2: UNTIL...END
```lulcode
UNTIL condition
  BTW loop body
END
```

**Pros**:
- Alternative to WHILE
- "Loop until condition becomes true"

**Cons**:
- Less common than WHILE

### Recommendation: **Support WHILE**

```lulcode
WHILE condition
  BTW body
END
```

**Transpilation**:
```lolcode
IM IN YR loop
  NOT condition, O RLY?
    YA RLY
      GTFO
  OIC
  BTW body
IM OUTTA YR loop
```

## Part 4: Loop Control

### BREAK (Exit Loop)
```lulcode
WHILE WIN                 BTW Infinite loop
  IF condition
    BREAK
  END
END
```

Transpiles to:
```lolcode
IM IN YR loop
  condition, O RLY?
    YA RLY
      GTFO
  OIC
IM OUTTA YR loop
```

### CONTINUE (Skip Iteration)
```lulcode
FOR i FROM 0 TO 10
  IF i == 5
    CONTINUE
  END
  VISIBLE i
END
```

**Problem**: LOLCODE has no CONTINUE!

**Solution**: Use conditional to skip body
```lolcode
i R 0
IM IN YR loop UPPIN YR i TIL BOTH SAEM i AN 10
  NOT BOTH SAEM i AN 5, O RLY?
    YA RLY
      VISIBLE i
  OIC
IM OUTTA YR loop
```

```lulcode

END
```

### LOOP (Infinite)
```lulcode

  END

```

### Loop Control
```lulcode
BREAK                     BTW Exit loop (transpiles to GTFO)
CONTINUE                  BTW Skip iteration (transpile to conditional)
```

## Edge Cases

### Nested Loops
```lulcode
FOR i FROM 0 TO 3
  FOR j FROM 0 TO 3
    VISIBLE "{i},{j}"
  END
END
```

Must generate unique loop labels:
```lolcode
i R 0
IM IN YR __for_loop_1 UPPIN YR i TIL BOTH SAEM i AN 3
  j R 0
  IM IN YR __for_loop_2 UPPIN YR j TIL BOTH SAEM j AN 3
    VISIBLE ":{i},:{j}"
  IM OUTTA YR __for_loop_2
IM OUTTA YR __for_loop_1
```

### BREAK in Nested Loop
```lulcode
FOR i FROM 0 TO 10
  IF i == 5
    BREAK                 BTW Breaks inner loop only
  END
END
```

### Variable Shadowing
```lulcode
VAR i = 100
FOR i FROM 0 TO 5
  VISIBLE i               BTW Uses loop i
END
VISIBLE i                 BTW Original i = 100 or 5?
```

**Decision**: Loop variable shadows outer scope (like most languages).

4. **Test LOOP**: Infinite with BREAK
5. **Test BREAK**: Exit loop
6. **Test CONTINUE**: Skip iteration (if implemented)
7. **Test nested loops**: Proper label generation
8. **Test FOREACH**: When arrays available
