# Story 003.5: Function Syntax Shorthand - Research & Design

**Status**: In Progress  
**Date**: 2026-05-19

## Overview

Consider cleaner function syntax. Current LOLCODE is extremely verbose: `HOW IZ I funcname YR arg1 AN YR arg2 ... IF U SAY SO`.

## Current LOLCODE Function Syntax

### Function Declaration
```lolcode
HOW IZ I greet YR name
  VISIBLE "Hello :{name}"
IF U SAY SO
```

### With Return Value
```lolcode
HOW IZ I add YR x AN YR y
  FOUND YR SUM OF x AN y
IF U SAY SO
```

### Multiple Parameters
```lolcode
HOW IZ I calculate YR a AN YR b AN YR c
  BTW Use a, b, c
  FOUND YR result
IF U SAY SO
```

### Function Call
```lolcode
I IZ greet YR "Alice" MKAY
I HAS A sum ITZ I IZ add YR 5 AN YR 10 MKAY
```

## Problem: Verbosity

- `HOW IZ I` = 3 words just to start function
- `YR` before each parameter
- `AN YR` between parameters
- `IF U SAY SO` = 4 words to end
- `I IZ ... MKAY` = 3 extra words per call

**Total**: ~10 extra tokens per function!

## Design Goals

1. **Reduce verbosity**: Shorter declaration syntax
2. **Clear parameters**: Easy to see argument list
3. **Return clarity**: Clear return statement
4. **Maintain some charm**: Not too generic
5. **Backward compatible**: HOW IZ I still works

## Function Declaration Options

### Option 1: FUNC keyword (C/Go-style)
```lulcode
FUNC greet(name)
  VISIBLE "Hello {name}"
END

FUNC add(x, y)
  RETURN SUM OF x AN y
END
```

**Pros**:
- Very concise
- Parentheses make parameters clear
- RETURN clearer than FOUND YR

**Cons**:
- Very generic (loses LOLCODE charm)
- Parentheses not LOLCODE-y

### Option 2: DEF keyword (Python/Ruby-style)
```lulcode
DEF greet(name)
  VISIBLE "Hello {name}"
END

DEF add(x, y)
  RETURN SUM OF x AN y
END
```

**Pros**:
- Short (3 chars)
- Common in dynamic languages
- Clear parameter list

**Cons**:
- Same issues as FUNC (generic)

### Option 3: Shortened LOLCODE
```lulcode
HOW greet YR name
  VISIBLE "Hello :{name}"
END

HOW add YR x AN YR y
  RETURN SUM OF x AN y
END
```

**Pros**:
- Preserves LOLCODE flavor (HOW)
- Just removes "IZ I" and "IF U SAY SO"
- Parameters still LOLCODE-y (YR, AN YR)

**Cons**:
- Still verbose (YR, AN YR)
- Only modest improvement

### Option 4: Hybrid
```lulcode
FUNC greet YR name         BTW FUNC + LOLCODE params
  VISIBLE "Hello :{name}"
END

FUNC add YR x AN YR y
  RETURN SUM OF x AN y
END
```

**Pros**:
- FUNC concise
- Params preserve LOLCODE flavor
- Balance of practical + whimsical

**Cons**:
- Inconsistent style (keyword modern, params old)

### Recommendation: **FUNC with Parentheses** (Optional Implementation)

```lulcode
FUNC greet(name)
  VISIBLE "Hello {name}"
END

FUNC add(x, y)
  RETURN SUM OF x AN y
END
```

**Rationale**:
- Maximum verbosity reduction
- Parentheses make parameter list unambiguous
- RETURN clearer than FOUND YR
- Can still use HOW IZ I for LOLCODE flavor

**Decision**: **Defer to future EPIC**

Functions are less commonly used than variables/loops/conditionals. Lower priority for MVP syntactic sugar.

## Function Call Options

### Current LOLCODE
```lolcode
I IZ funcname YR arg1 AN YR arg2 MKAY
```

### Option 1: Standard Call Syntax
```lulcode
funcname(arg1, arg2)
```

**Pros**:
- Universal, everyone knows it
- Concise
- Clear argument list

**Cons**:
- Generic (loses LOLCODE charm)

### Option 2: Shortened LOLCODE
```lulcode
CALL funcname YR arg1 AN YR arg2
```

**Pros**:
- Preserves LOLCODE style
- CALL explicit

**Cons**:
- Still verbose
- Minimal improvement

### Recommendation: **Support Both**

```lulcode
greet("Alice")            BTW Modern style
I IZ greet YR "Alice" MKAY   BTW LOLCODE style
```

Both transpile to:
```lolcode
I IZ greet YR "Alice" MKAY
```

## Return Statement

### Current LOLCODE
```lolcode
FOUND YR value
```

### Option 1: RETURN keyword
```lulcode
RETURN value
```

**Pros**:
- Universal
- Clear intent
- Short

**Cons**:
- Generic

### Option 2: Keep FOUND YR
```lulcode
FOUND YR value            BTW "I found your value"
```

**Pros**:
- Preserves LOLCODE charm
- Unique, memorable

**Cons**:
- Longer
- Less familiar

### Recommendation: **Support Both**

```lulcode
RETURN value              BTW Modern
FOUND YR value            BTW LOLCODE
```

## Complete Syntax Specification (Deferred)

### Function Declaration
```lulcode
FUNC name(param1, param2)
  BTW body
  RETURN value
END
```

Transpiles to:
```lolcode
HOW IZ I name YR param1 AN YR param2
  BTW body
  FOUND YR value
IF U SAY SO
```

### Function Call
```lulcode
result = funcname(arg1, arg2)
```

Transpiles to:
```lolcode
result R I IZ funcname YR arg1 AN YR arg2 MKAY
```

### Backward Compatibility
```lolcode
HOW IZ I oldStyle YR param
  FOUND YR param
IF U SAY SO
```

Still works unchanged.

## Examples (Future)

### Example 1: Simple Function
```lulcode
FUNC greet(name)
  VISIBLE "Hello {name}!"
END

greet("Alice")
```

### Example 2: With Return
```lulcode
FUNC add(x, y)
  RETURN SUM OF x AN y
END

VAR sum = add(5, 10)
VISIBLE "Sum: {sum}"
```

### Example 3: Multiple Parameters
```lulcode
FUNC format(title, name, age)
  RETURN "{title} {name}, age {age}"
END

VAR greeting = format("Dr.", "Smith", 45)
VISIBLE greeting
```

## Transpilation Strategy (Future)

### Parse Function Declaration
```javascript
// FUNC name(params) ... END
output = output.replace(
  /FUNC\s+(\w+)\(([^)]*)\)\n([\s\S]+?)END/g,
  (match, name, params, body) => {
    const lolParams = params.split(',')
      .map(p => `YR ${p.trim()}`)
      .join(' AN ');
    return `HOW IZ I ${name} ${lolParams}\n${body}IF U SAY SO`;
  }
);
```

### Parse Function Call
```javascript
// funcname(args)
output = output.replace(
  /(\w+)\(([^)]*)\)/g,
  (match, name, args) => {
    const lolArgs = args.split(',')
      .map(a => `YR ${a.trim()}`)
      .join(' AN ');
    return `I IZ ${name} ${lolArgs} MKAY`;
  }
);
```

### Parse RETURN
```javascript
// RETURN value
output = output.replace(/RETURN\s+(.+)/g, 'FOUND YR $1');
```

## Impact Analysis (Future)

### Before
```lolcode
HOW IZ I add YR x AN YR y
  I HAS A result ITZ SUM OF x AN y
  FOUND YR result
IF U SAY SO

I HAS A sum ITZ I IZ add YR 5 AN YR 10 MKAY
VISIBLE sum
```

**6 lines, 35+ tokens**

### After
```lulcode
FUNC add(x, y)
  VAR result = SUM OF x AN y
  RETURN result
END

VAR sum = add(5, 10)
VISIBLE sum
```

**6 lines, 20 tokens** (43% reduction!)

### Variadic Functions (Future Future)
```lulcode

  BTW Sum all numbers
END
```

## Decision: **DEFER FUNCTION SYNTAX**

**Rationale**:
- Functions less common than vars/loops/conditionals in typical code
- Lower priority for MVP
- More complex transpilation (need to parse full function bodies)
- Can be added in future EPIC

**For MVP**: Focus on high-value sugar:
1. ✅ VAR (variable declarations)
2. ✅ IF/ELIF/ELSE (conditionals)
3. ✅ = operator (assignment)
4. ✅ FOR/WHILE/LOOP (loops)
5. ❌ FUNC (defer to EPIC-019)
