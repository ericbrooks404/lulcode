# Story 005.1: CLI Interface Design

**Status**: In Progress  
**Date**: 2026-05-19

## Overview

Design a professional command-line interface for LULCODE that makes it easy to transpile, run, and manage LULCODE programs.

## Current CLI

**What we have** (`src/transpiler.js`):
```bash
lulcode input.lul [output.lol]
lulcode --help
lulcode --version
```

**Limitations**:
- No subcommands (just one mode)
- No `--run` option (must manually run LCI)
- No `--watch` mode
- No colored output
- Basic error handling
- No project initialization

## Enhanced CLI Design

### Command Structure

```bash
lulcode <command> [options] [file]
```

### Commands

#### 1. `lulcode compile` (or just `lulcode`)
Transpile LULCODE to LOLCODE

```bash
lulcode compile input.lul              # Output to stdout
lulcode compile input.lul -o output.lol  # Output to file
lulcode compile input.lul --run          # Transpile and run with LCI
lulcode input.lul -o output.lol          # Shorthand (compile is default)
```

**Options**:
- `-o, --output <file>` - Output file (default: stdout)
- `-r, --run` - Execute with LCI after transpiling
- `-w, --watch` - Watch for changes and re-transpile
- `-v, --verbose` - Show debug output
- `--lci-path <path>` - Custom LCI executable location

#### 2. `lulcode run`
Transpile and execute in one step

```bash
lulcode run program.lul                  # Transpile + execute
lulcode run program.lul -- arg1 arg2     # Pass args to program
```

**Behavior**:
- Transpiles to temporary file
- Executes with LCI
- Cleans up temp file
- Shows both transpile and runtime output

**Options**:
- `--keep` - Keep transpiled .lol file
- `-v, --verbose` - Show transpilation details
- `--lci-path <path>` - Custom LCI path

#### 3. `lulcode init`
Initialize a new LULCODE project

```bash
lulcode init [directory]                 # Create new project
lulcode init                            # Init in current directory
```

**Creates**:
```
my-project/
├── main.lul          # Hello world program
├── .gitignore        # Ignore .lol files
└── README.md         # Project template
```

**Options**:
- `--template <name>` - Use template (basic, game, cli, etc.)

#### 4. `lulcode check`
Validate LULCODE syntax without transpiling

```bash
lulcode check program.lul                # Check syntax
lulcode check **/*.lul                   # Check all .lul files
```

**Output**:
- ✅ Syntax OK
- ❌ Syntax errors with line numbers and helpful messages

#### 5. `lulcode version`
Show version information

```bash
lulcode version                          # Show LULCODE version
lulcode -v                               # Shorthand
lulcode --version                        # Longhand
```

**Output**:
```
LULCODE v0.2.0
Transpiler: v0.2.0
LCI: v0.11.2 (bundled)
Node.js: v18.0.0
```

#### 6. `lulcode help`
Show help information

```bash
lulcode help                             # General help
lulcode help compile                     # Command-specific help
lulcode --help                           # Shorthand
```

#### 7. `lulcode doctor`
Check installation and dependencies

```bash
lulcode doctor                           # Diagnose issues
```

**Checks**:
- ✅ LULCODE transpiler found
- ✅ LCI executable found at ~/.lulcode/bin/lci
- ✅ Node.js version compatible
- ❌ Missing dependencies → suggests fixes

### Global Options

Options that work with any command:

- `-h, --help` - Show help
- `-v, --version` - Show version
- `--verbose` - Verbose output
- `--quiet` - Suppress non-error output
- `--no-color` - Disable colored output

### Examples

```bash
# Quick transpile and run
lulcode run myprogram.lul

# Transpile for distribution
lulcode compile app.lul -o app.lol

# Development mode with auto-reload
lulcode compile app.lul --watch --run

# Check syntax before commit
lulcode check src/**/*.lul

# Start new project
lulcode init my-game

# Debug transpilation
lulcode compile program.lul --verbose

# Use custom LCI
lulcode run program.lul --lci-path /usr/local/bin/lci

# Pipe from stdin
cat program.lul | lulcode > output.lol
```

### Colored Output

Use chalk or similar for colored output:

```
✅ Transpiled main.lul → main.lol (0.05s)
🚀 Running with LCI...
───────────────────────────────
[program output]
───────────────────────────────
✅ Execution completed (0.12s)
```

**Error output**:
```

  13 | VAR x = 10
  14 | IF x == 10
> 15 |   VISIBLE x
     |   ^ Expected END
  16 | VAR y = 20
  
Tip: IF statements must end with END keyword
```

### Progress Indicators

For long operations:
```
Transpiling program.lul...
├─ Parsing...       ✅ (0.01s)
├─ Transforming...  ✅ (0.03s)
└─ Writing output... ✅ (0.001s)

✅ Done in 0.04s
```

## Error Handling

### Friendly Error Messages

**File not found**:
```
❌ Error: Could not find 'program.lul'

Did you mean one of these?
  • program.lol
  • program-old.lul
  • test-program.lul
```

**LCI not found**:
```
❌ Error: LCI interpreter not found

LULCODE requires LCI to run programs.

To install LCI:
  lulcode install-lci

Or download manually from:
  https://github.com/justinmeza/lci
```

**Syntax error**:
```
❌ Transpilation failed

program.lul:23:5
  21 | FOR i FROM 0 TO 10
  22 |   VISIBLE i
> 23 | ENDFOR
       ^^^^^ Unexpected token
  24 | 
  
Expected: END
Got: ENDFOR

Tip: FOR loops end with END, not ENDFOR
```

## Watch Mode

```bash
lulcode compile program.lul --watch --run
```

**Behavior**:
1. Initial transpile + run
2. Watch for file changes
3. On change: re-transpile + re-run
4. Show minimal diff output

**Output**:
```
🔍 Watching program.lul for changes...
✅ Transpiled and running...

[program output]

───────────────────────────────
👀 Watching for changes... (Ctrl+C to stop)

[file changed detected]

🔄 Changes detected, reloading...
✅ Transpiled and running...

[new program output]
```

## Implementation Strategy

### CLI Framework: Commander.js

**Why Commander**:
- Most popular Node.js CLI framework
- Clean API for commands and options
- Good help generation
- Subcommand support

**Example**:
```javascript
const { Command } = require('commander');
const program = new Command();

program
  .name('lulcode')
  .description('LULCODE to LOLCODE transpiler')
  .version('0.2.0');

program
  .command('compile <file>')
  .description('Transpile LULCODE to LOLCODE')
  .option('-o, --output <file>', 'output file')
  .option('-r, --run', 'execute with LCI')
  .option('-w, --watch', 'watch for changes')
  .action(compile);

program
  .command('run <file>')
  .description('Transpile and execute')
  .action(run);

program.parse();
```

### Color/Styling: Chalk

```javascript
const chalk = require('chalk');

console.log(chalk.green('✅ Success!'));
console.log(chalk.red('❌ Error!'));
console.log(chalk.yellow('⚠️  Warning!'));
console.log(chalk.dim('debug info'));
```

### Progress: Ora

```javascript
const ora = require('ora');

const spinner = ora('Transpiling...').start();
// do work
spinner.succeed('Transpiled successfully!');
```

### Watch: Chokidar

```javascript
const chokidar = require('chokidar');

chokidar.watch('*.lul').on('change', (path) => {
  console.log(`File ${path} changed, recompiling...`);
  compile(path);
});
```
